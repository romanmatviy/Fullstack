Backend development

I develop reliable and scalable server code on Laravel (PHP 8+). REST API, authentication, queues, task scheduler, integration with any services.

Directions of backend development

Laravel is my primary framework for back-end development. I have deep knowledge of its ecosystem: Eloquent ORM, Queues, Events, Broadcasting, Policies, Sanctum/Passport for authentication and many more.

I write clean, documented and tested code. I follow SOLID principles, I use Design Patterns where it is justified — Repository, Service Layer, DTO, Observer.

What I do on the backend

  • ✓
    REST API — design and development of a full-fledged API with documentation (OpenAPI/Swagger), versioning and correct HTTP statuses.
  • ✓
    Authentication and authorization — Laravel Sanctum, Passport, JWT, OAuth2, roles and permissions.
  • ✓
    Queues and background tasks — processing of email, SMS, reports, import/export via Laravel Queue + Redis/Database.
  • ✓
    Integration — payment gateways, Nova Poshta, Telegram Bot, Google APIs, SMS, CRM systems.
  • ✓
    Databases — design of MySQL/PostgreSQL schemas, migrations, query optimization, indexes.
  • ✓
    Testing — PHPUnit tests, Feature and Unit tests for critical logic.

For which projects

Backend development is required for any project where there is data and logic: online stores, SaaS platforms, mobile applications (backend for iOS/Android), corporate systems, API microservices.

I am also ready to join the existing Laravel project: conduct a code review, refactoring, optimization or add new functionality.

Approximate terms

  • Complex backend1–3 months
  • Refactoring / auditNegotiable

Frequently asked questions

What is a REST API and why do I need it?

The REST API is the "bridge" between your site/application and the database. It allows mobile applications, frontend and other systems to exchange data.

Can you join an existing project?

Yes, I work with both new and existing Laravel projects — code review, refactoring, adding functionality.

Do you write tests?

Yes, I cover the critical business logic of PHPUnit with tests (Feature and Unit), which ensures the stability of the code.

What databases do you support?

MySQL, PostgreSQL, Redis (for caching and queues), MongoDB — the choice depends on the project tasks.
